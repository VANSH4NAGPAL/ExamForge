# ⚙️ ServiceNow – Workflow Studio (Study Notes)

---

## 1. What is Workflow Studio?

- Centralized environment for **designing, managing & automating workflows**
- Visual drag-and-drop interface
- Connects: tasks, approvals, notifications, conditions, external systems

### Key Components:
| Component | Purpose |
|-----------|---------|
| **Stages** | Represent progress of a workflow |
| **Activities** | Specific actions/tasks to complete |
| **Triggers** | Start a flow automatically based on events |
| **Conditions** | Control flow based on user input or system data |

---

## 2. Flows vs Traditional Workflows

| | Traditional Workflows | Flows (Modern) |
|-|----------------------|----------------|
| **Status** | ⚠️ Being **deprecated** | ✅ Preferred going forward |
| **Future dev** | No new features planned | Active development |
| **Interface** | Older | Drag-and-drop, visual |
| **Logic** | Static | Dynamic (conditions + branching) |
| **Still used for** | Knowledge Management (2 workflows) | Service Catalog, most everything else |

> ⚠️ Workflows still supported but **no new improvements planned** — migrate to Flows!

---

## 3. Trigger Types

| Trigger Type | When it fires | Extra |
|-------------|--------------|-------|
| **Record-based** | When a record is created, updated, or deleted | Triggering record usable as input later in flow |
| **Date-based** | At a specific date/time (daily, weekly, monthly, custom) | Execution time usable as input |
| **Application-based** | When a specific app spoke is activated | May require plugin activation |

### Creating a Trigger:
- Workflow Studio homepage → **New → Trigger**
- **Action controls**: undo/redo changes
- **View**: preview latest changes
- **Publish**: makes trigger available to users

---

## 4. Spokes

- A **spoke** = bundle of triggers & actions for a specific application
- Activated when the **parent application is activated**
- Example: **ITSM spoke** → includes "Create Task" action
- Extra spokes available in **ServiceNow Store** (filtered by version & dependencies)

---

## 5. Core Actions (Examples)

| Action | What it does |
|--------|-------------|
| **Ask for Approval** | Creates approvals with rules for approve/reject/cancel + due date |
| Others | Notifications, record updates, subflow calls, etc. |

> Additional actions unlocked by activating **application-specific spokes**

---

## 6. Data Pills & Data Section

- **Data Pills** = pieces of data generated by triggers or actions in a flow
- Used as **inputs for subsequent actions, flows, or subflows**
- Add them by: **drag & drop** from Data section OR use the **Data Pill Picker icon**
- Use **sequence values** (Trigger 1, 2, 3…) to identify the correct pill
- Dot-walk using **arrow keys** in Data Pill Picker to access fields from related tables
- Runtime values stay **stable** throughout the flow

---

## 7. Flow Design Tips

- ✅ Workflow Studio **auto-saves** progress — safe even if internet drops
- Test → Save → **Activate** to make flow operational
- Results of one action = input for the next
- Flows integrate with: Service Catalog, incident resolution, procurement, and more

---

## 8. Integration Hub

- Enables **integrations with external systems** using common protocols
- Developers can build **custom integrations** → packaged as reusable actions
- Use pre-built ServiceNow components OR fully custom integrations
- Supports: data transfers, external API triggers, record sync
- ⚠️ Requires a **separate subscription** — check licensing first!

---

## 9. Playbooks

- Automate & streamline processes using existing ServiceNow capabilities
- Can automate: **notifications, record creation, approvals, and more**
- Built on **same technology as Workflow Studio** → integrates with flows, actions & flow executions
- Guide users through processes with **consistency & efficiency**

---

> **Key Exam Points:**
> - Traditional workflows = **deprecated** (no new features) — Flows are the future
> - Knowledge Management still uses **workflows** (not flows)
> - 3 trigger types: **Record-based, Date-based, Application-based**
> - **Spoke** = triggers + actions bundled for a specific app
> - Data pills = **drag & drop** or use **Data Pill Picker**
> - Use **sequence values** to identify correct data pill
> - Dot-walking in Data Pill Picker = **arrow keys**
> - Integration Hub = **separate subscription required**
> - Playbooks = built on same tech as Workflow Studio
